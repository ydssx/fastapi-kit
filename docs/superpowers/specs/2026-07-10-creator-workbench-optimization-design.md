# 创作者工作台优化设计

**日期：** 2026-07-10  
**范围：** `creator/` 中已登录创作工作台；不包含登录、找回密码和重置密码页面。

## 目标

将当前的暗色内容工作室从“能完成创作步骤的工具集合”优化为一条可信、可推进、可回溯的创作主线：

1. 用户打开项目页即可识别最需要推进的工作与阻塞项。
2. AI 建议是可比较、可定位插入、可撤销的编辑协作，而不是脱离编辑器的结果展示。
3. 发布前的遗漏在发起完成操作前即可被看见与解决。
4. 灵感、项目和品牌约束使用统一的中文术语、状态与反馈方式。
5. 键盘、屏幕阅读器、移动端和减少动态偏好均能完成核心流程。

## 已确认的产品与视觉方向

采用“发布节奏中枢”为主、“人机共写台”为核心交互、“连续工作室体验”为统一规范的组合。

- 保留项目首页的发布冲刺优先级，而不退回按时间排序的项目列表。
- 保留固定侧栏、暗色内容工作室、珊瑚／薄荷双色和现有流水线概念。
- 珊瑚色只代表需要行动；薄荷色只代表 AI 或已就绪；黄色只代表风险。状态色必须同时有文字与图标说明。
- `Fraunces` 用于页面命题与关键项目名，`Outfit` 用于正文与操作，`JetBrains Mono` 用于配额、阶段、来源和状态。
- 现有 Stage Line 演化为“行动信号线”：依据页面中的风险或 AI 状态呈现，但不替代文本状态。

## 页面与交互设计

### 项目首页

- 页面标题改为面向行动的“今天的创作队列”，并展示需要注意的项目数量。
- 焦点冲刺卡始终是页面唯一主操作：展示项目名、下一步、发布阻塞数，以及“继续完成”按钮。
- 后续项目按“继续创作”和“发布检查”组织。每一项仅展示下一步、状态和必要的风险，不重复所有项目元数据。
- 创建项目空态不再指向“左侧表单”；在移动端使用“创建项目”或“前往灵感实验室”的真实可见操作。
- 窄屏将侧栏替换为已有移动顶栏，并将焦点冲刺卡扩展至全宽。

### 项目详情与 AI 共写

- 编辑器保存可恢复的 DOM 选区或文本偏移量。AI 的“插入选区”只在选区有效时可用；无选区时显示“插入到末尾”这一准确文案。
- AI 面板继续使用 A/B/C 变体，但提供明确的“预览”“替换选区”“插入选区”“采纳全部”操作。采纳后更新草稿，并保留步骤版本历史作为回溯路径。
- 生成期间显示明确的等待状态、正在使用的品牌约束和风险信息；不暗示生成结果可直接发布。
- 项目标题输入框恢复全局 `:focus-visible` 焦点轮廓，不能以弱底边框取代。

### 发布检查

- 当存在待完成发布项时，“完成项目”按钮禁用，并在其附近列出剩余项数和未完成平台。
- 所有清单项完成后才启用按钮；提交中保持禁用并显示“提交中…”。
- 后端完成校验仍保留，以处理并发更新或客户端绕过。

### 灵感实验室与交接

- 用户界面统一使用中文术语：“优化想法”替代 Refine，“交接到项目”替代 handoff，“发布检查”替代 checklist。
- 会话提示改为“关闭此标签页可能丢失未交接的内容”，与 `sessionStorage` 行为一致。
- 交接弹窗列出即将注入项目的主题、目标平台、大纲和初始步骤，要求用户确认后创建项目。
- 弹窗补齐初始聚焦、焦点锁定、Escape 关闭、触发元素焦点恢复与背景点击关闭。

### 品牌档案与全站反馈

- 品牌档案仍集中编辑；生成和发布区域仅显示紧凑的“品牌约束来源”与必要风险。
- 保存成功、密码成功等非错误反馈使用 `role="status"` 或适当的 `aria-live="polite"`。
- 移动导航补齐 `aria-current="page"`；桌面与移动导航使用同一活动状态判定。
- 取消依赖原生确认对话框的关键创作操作，改用项目一致的确认界面；本次优先覆盖灵感交接，其他原生确认按现有范围逐步替换。

## 组件与边界

- 继续使用 React、React Router、TanStack Query、CSS Modules 和既有 API 客户端；不引入组件库或新状态管理框架。
- 将项目首页的行动焦点、项目行状态、发布阻塞提示提取为可复用展示组件；排序仍由现有 `buildProjectPriorityView` 决定。
- 项目详情中的编辑器选区适配层仅负责捕获／恢复选区和将文本操作映射为草稿更新；AI 请求和版本保存继续留在现有页面与 API 边界。
- 把 `PipelineThumbnail` 的硬编码颜色转换为语义 token；保留每类缩略图的独特色相和识别性。
- 不修改 Creator API 契约、数据库 schema 或认证页。

## 错误与状态

- AI 请求失败、配额耗尽、草稿保存失败沿用现有 API 错误处理，并在操作所在区域提供明确可恢复提示。
- 选区在 AI 请求期间失效时，不执行不准确的插入；提示用户重新选择文本或改为插入末尾。
- 发布清单更新失败时保持服务器状态为准，并向用户显示失败原因。
- 所有通知的错误使用 `role="alert"`，成功和一般状态使用 `role="status"`。

## 验证标准

### 自动验证

- 为选区插入、无选区降级、发布完成禁用、交接弹窗焦点行为添加或扩展前端测试。
- 运行 Creator 前端 lint 与生产构建。

### 手工验证

- 桌面、900px 断点与窄屏下，项目首页的焦点任务、创建项目入口和移动导航均可见且语义正确。
- 全键盘流程可进入并退出交接弹窗，焦点不会逸出，关闭后回到触发按钮。
- 使用屏幕阅读器时，页面状态、保存成功和错误反馈都可播报。
- 选择 AI 文本后能够正确插入或替换；无选区时文案和行为一致。
- 发布项未完成时不能完成项目；完成后可以提交。
- 启用减少动态后，行动信号线与过渡不产生不必要动画。

## 非目标

- 重做登录及密码恢复流程。
- 变更项目排序、发布规则或 AI 模型／提示词策略。
- 持久化 Playground 会话到后端。
- 引入日历排期、协作成员或新的项目数据字段。
