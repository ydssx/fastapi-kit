Grounding Context

Codebase Context — FastAPI + React creator 工作台（/creator/），流水线向导 + 品牌档案 + 步骤级 AI 建议（AiSuggestionPanel、ContextChips）。当前 AI 仅绑定「当前步骤」、返回单一建议；第 3 步起空编辑器自动拉建议。已有 artifact 版本化、open_step 回开 remix、pipeline 模板定义、admin creator-metrics。

  • 痛点：空白页焦虑、上下文在 ChatGPT/表格间断裂；无灵感画廊/模板市场；单方案生成限制探索。

  • 外部：Jasper Canvas、Canva Magic Switch、Copy.ai 工作流、DAW 采样库、Figma 社区、厨师 mise en place、Writer Semrush 趋势侧栏。

#1

并排多方案面板（N-Choice Panel）

Description将 ai-suggest 从返回单一 suggestion 扩展为 3 个并排卡片（不同角度/语气），用户可对比、采纳其一或合并。UI 复用现有 AiSuggestionPanel 的 adopt/insert/regenerate 模式，后端一次 LLM 调用要求 JSON 数组或并行 3 次低温/高温采样。

AxisAI 建议与多方案生成

Basisdirect: creator/src/constants.ts 中 topic/hook 步已有 chip「再给 3 个角度」；app/services/creator_ai.py 仅返回单条 suggestion；前端「换一版」是串行而非并排对比。

Rationale产品已承认用户需要多角度，但 UX 仍是一次只看一版——并排是「获取灵感」最低成本、最高感知价值的改动，直接消减「换一版直到满意」的配额消耗与挫败感。

Downsides配额计次需定义（1 次调用 vs 3 次）；移动端三栏需折叠为 swipe；prompt 需约束多样性避免三版雷同。

Confidence92% High

ComplexityMedium

#2

选题 Playground（脱离步骤的 ideate 模式）

Description新增独立入口「灵感实验室」：不绑定 current_step_key，用户输入模糊方向或粘贴参考链接，系统基于品牌档案返回 5–10 条选题/钩子/角度清单，选中后再「落地为项目」。API：POST /api/v1/creator/ai/ideate，与步骤内 ai-suggest 分离。

Axis探索性工作流

Basisdirect: creator_ai.py 校验 step_key == project.current_step_key；brainstorm 文档指出「上下文断裂、在各工具间复制粘贴」是核心痛点；当前只能在流水线内逐步获取建议。

Rationale灵感获取发生在「决定做什么」之前，而现有产品把 AI 锁在「正在做哪一步」——Playground 对齐用户真实心智：先发散，再进入流水线。

Downsides与流水线状态可能脱节，需「从 Playground 创建项目」的明确 handoff；额外配额品类。

Confidence88% High

ComplexityMedium

#3

灵感采样库（Inspiration Sample Rack）

Description按「语气 × 平台 × 内容类型」预置可拖拽的灵感片段（钩子句式、标题公式、结构骨架）——像 DAW 采样库，用户拖入编辑器或作为 adjustment 发送给 AI。首版可用静态 JSON + 品牌过滤，无需 LLM。

Axis冷启动与空白页灵感

Basisexternal: 音乐 DAW sample library 模式（按情感/节奏组织）；direct: 原型设计文档明确「减少空白页焦虑」为方案 C 动机；现有空态仅有「新建流水线」引导，无内容级 spark。

Rationale不是所有灵感都需要生成——结构化片段让用户「先动手再优化」，比等待 AI 生成更快打破空白页，且不消耗配额。

Downsides内容库维护成本；需与品牌 taboo 过滤；国际化/行业垂直需扩展库。

Confidence85% High

ComplexityMedium

#4

Mise en Place 准备台（动笔前的备料页）

Description在进入第 3 步（脚本/正文）前，插入轻量「备料」屏：自动汇总品牌档案摘要、可选竞品 URL（提取要点）、3 条平台热点 chip——全部进入 ContextChips 与后续 prompt。用户确认「备料完成」再进入 auto-suggest 流程。

Axis品牌与上下文驱动灵感

Basisexternal: 厨师 mise en place（动笔前备好料）；direct: shouldAutoSuggest 从 step index ≥ 2 触发，但此前两步用户可能未建立足够上下文；ContextChips 仅展示品牌三字段+上步摘要，无外部信号。

Rationale灵感质量高度依赖输入上下文——在 auto-suggest 触发点之前系统化「备料」，比事后用 adjustment chip 补丁式补救更根本。

Downsides多一步 friction；URL 抓取需新依赖与合规；热点源需选型（手动输入 vs API）。

Confidence80% Med-High

ComplexityMedium

#5

灵感画廊 + 一键 Remix

Description在项目列表旁增加「灵感画廊」：展示 docs/prototypes/v2/ 截图语义、高完成率 pipeline 示例（来自 creator-metrics）、用户自己的已完成项目摘要。点击「Remix」→ 新建项目并预填首步草稿（复用 open_step / artifact 复制逻辑）。

Axis发现与复用

Basisdirect: creator/docs/prototypes/ 已有 v1/v2 视觉资产；open_step + artifact 版本链已在测试中验证 remix 路径；external: Canva Magic Switch 一键重混模式。

Rationale「看别人怎么做」是最古老的灵感来源——仓库里已有原型与版本化产物，只差发现层 UI。

Downsides隐私：默认仅展示用户自己的完成项目 + 官方 curated；metrics 聚合需脱敏；画廊内容运营。

Confidence84% High

ComplexityMedium

#6

可 Fork 的流水线 / 提示词模板库

Description将 app/creator/pipelines.py 中的流水线定义 + app/creator/prompts/ 模板暴露为可浏览、可 Fork 的「社区模板」——用户 Fork 后创建项目，prompt 模板带变量占位符（受众、话题）。高完成率模板由 metrics 加权排序。

Axis发现与复用

Basisdirect: pipeline 以代码常量存在，已是模板形态；admin GET /api/v1/admin/creator-metrics/summary 可识别高频 pipeline；external: Musebox / Figma Community 的 Fork 模式。

Rationale降低「从哪开始」的决策成本——模板是结构化的灵感，比纯 AI 生成更可预测、更可复用。

Downsides社区治理、模板质量审核；从代码常量到用户可见库需数据模型；UGC 模板远期工作量大。

Confidence76% Med

ComplexityHigh

#7

趋势-to-Brief 侧边栏（Trend Chips）

Description在 StepWorkspace 右侧或 ContextChips 行扩展「今日热点」chip（按目标平台过滤），点击将趋势词注入 adjustment 或 Playground prompt。首版可手动 curated + 定期更新，后续接 Semrush/平台 API。

Axis冷启动与空白页灵感

Basisexternal: Writer.com Semrush 连接器将 SEO 趋势直接变为 Brief；direct: ContextChips 当前仅品牌+上步，无外部市场信号入口。

Rationale创作者常见阻塞是「不知道写什么」——趋势 chip 把外部 demand signal 变成可点击的灵感起点。

Downsides趋势 API 成本与准确性；可能与品牌 taboo 冲突；需明确非实时时的 stale 提示。

Confidence70% Med

ComplexityHigh
