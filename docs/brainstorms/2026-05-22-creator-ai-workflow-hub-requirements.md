# 多平台创作者 AI 创作流水线工作台

## Summary

面向多平台个人创作者的 C 端产品：用少量固定流水线把「选题→脚本→制作准备→发布清单」串在同一工作台里，减少在多个 AI 工具间来回切换；第一版以流程编排与品牌记忆为主，不追求多平台自动发帖与全平台数据大屏。

---

## Problem Frame

多平台创作者在规划与生产内容时，常见做法是在 ChatGPT、各类生成工具、表格/笔记之间反复复制粘贴：上下文断裂、版本对不上、不知道每条内容卡在哪一步。通用项目管理工具不理解「脚本/分镜/封面/排期」等创作语义；纯 AI 聊天又没有流水线状态，导致断更、漏发或同一灵感在各平台各做一遍、风格不一致。调研显示「选题与内容流水线」与「多 AI 工具拼凑」是高频讨论点，但当前对话尚未用一手创作者访谈验证付费意愿。

---

## Actors

- A1. **多平台个人创作者**：在 2 个及以上内容平台经营账号，自己完成或主导策划、写稿、制作与发布节奏。
- A2. **产品/运营（内部）**：通过访谈与试用数据验证方向，不面向 C 端暴露。

---

## Key Flows

- F1. **从灵感到流水线项目**
  - **Trigger：** 用户有一个新选题或灵感，想做成可执行的一条内容。
  - **Actors：** A1
  - **Steps：** 选择内容类型（对应某条固定流水线）→ 录入或生成选题要点 → 系统创建「内容项目」并进入第一步 → 按步骤推进（提纲/脚本/制作清单等依流水线定义）→ 每步产出保留版本并进入下一步上下文。
  - **Outcome：** 单条内容在系统内有清晰阶段与可回溯产出，无需在外部工具间手动拼上下文。
  - **Covered by：** R1, R2, R3, R4

- F2. **带品牌记忆的 AI 辅助**
  - **Trigger：** 用户在流水线某步需要 AI 生成或改写。
  - **Actors：** A1
  - **Steps：** 系统加载用户配置的品牌/人设约束（语气、禁忌、结构偏好）→ 用户触发该步 AI 辅助 → 产出写入当前步骤并可编辑 → 用户确认后进入下一步或保存草稿。
  - **Outcome：** 生成内容默认符合其人设，且与当前项目上下文一致。
  - **Covered by：** R5, R6

- F3. **发布前核对（无自动发帖）**
  - **Trigger：** 内容进入「可发布」阶段。
  - **Actors：** A1
  - **Steps：** 查看各目标平台的发布 checklist（标题变体、比例提醒、文案槽位等抽象项，非 API 发帖）→ 用户手动标记各平台「已发布」或记录发布时间 → 项目归档或进入历史。
  - **Outcome：** 用户清楚哪些平台已发、哪些未发，避免漏发；系统不代为调用平台接口。
  - **Covered by：** R7, R8

- F4. **验证期访谈与试用（产品验证，非 C 端功能）**
  - **Trigger：** 内测或早期公测阶段。
  - **Actors：** A1, A2
  - **Steps：** 招募 5–10 名目标创作者 → 完成至少 1 条完整流水线 → 收集「是否愿每周使用 / 是否愿付费」反馈 → 决定是否扩展流水线或加码数据反馈。
  - **Outcome：** 在扩功能前获得真实行为证据，而非仅依赖行业文章。
  - **Covered by：** R15（成功标准中的验证门槛）

## Requirements

**流水线与项目**
- R1. 第一版须提供 **2 条固定流水线**：**短视频（偏制作）** 与 **长图文（偏传播）**，每条包含有序阶段；用户不能从零自定义无限步骤（避免做成通用 BPM）。
- R1a. **短视频流水线**须至少包含：选题 → 钩子/提纲 → 口播脚本 → 分镜/画面要点 → 封面/标题 → 素材清单或 B-roll/剪辑备注 → 发布核对（共 7 步，允许合并展示但不可省略「制作向」环节）。
- R1b. **长图文流水线**须至少包含：选题 → 大纲 → 正文 → 标题/摘要 → 配图要点 → 关键词/话题标签/多平台标题变体 → 发布核对（共 7 步）。
- R2. 用户可从选题/灵感创建一条 **内容项目**，项目须展示当前阶段、历史阶段产出与最后更新时间。
- R3. 用户在任一步骤的产出须可 **编辑与保存**；进入下一步时，上一步确认后的内容作为后续 AI 与展示的默认上下文。
- R4. 用户须能查看单项目的 **版本历史**（至少保留每步最近一次确认版；不要求完整 diff 引擎）。

**品牌与人设记忆**
- R5. 用户可配置 **品牌/人设档案**（如语气、受众、禁忌词、常用结构），并在流水线 AI 步骤中默认应用。
- R6. 用户可 **跳过或关闭** 某步 AI 建议，仅用手动输入完成该步，不强制每步都调用 AI。

**多平台（抽象，无 API）**
- R7. 项目须支持标记 **目标平台槽位**（预设中文生态常见平台名称或用户自选标签），用于发布 checklist，不要求对接官方发布 API。
- R8. 发布 checklist 须列出与各平台相关的 **提醒项**（如标题长度、竖横版、话题标签等——以产品文案配置为主），用户手动勾选完成状态。

**账号与商业化（C 端基础）**
- R9. 用户须能 **注册/登录** 并仅访问自己的项目与品牌档案（个人账户隔离）。
- R10. 须提供 **免费档与付费档** 的混合用量边界：**每月 2 个完整走完流水线的项目**（免费档）+ **AI 辅助次数月度上限**；超出任一项时明确提示升级，不在静默中扣费。Pro 档解除或提高上述上限（具体数值留 planning 配置）。

**验证与度量（面向 A2）**
- R11. 产品须能观测：**创建项目数、完成整条流水线的项目数、7 日内回访**，用于验证期决策（不要求对外展示 BI）。

## Acceptance Examples

- **Covers R1, R2.** 用户选择「短视频」流水线并输入选题「初夏穿搭 3 件套」→ 系统创建项目，当前阶段为「选题确认」，列表中可见该项目。
- **Covers R3, R4.** 用户在「脚本」步生成并编辑脚本后点击确认进入「制作清单」→ 「脚本」步显示为已完成，且可回看已确认脚本内容，不可 silently 丢失。
- **Covers R5, R6.** 用户品牌档案设定「口语化、禁用夸张承诺」→ 在脚本步触发 AI → 产出无明显违反禁忌的文案；用户选择「本步不用 AI」手动粘贴脚本 → 仍可确认并进入下一步。
- **Covers R7, R8.** 用户为目标平台勾选「小红书」「公众号」→ 进入发布核对页看到两平台 checklist → 手动标记小红书已发布、公众号未发布 → 项目状态反映「部分已发」。
- **Covers R10.** 免费档用户本月已完成允许数量的完整项目 → 再创建时提示升级或下月再试，且不扣除未说明的额外费用。

## Success Criteria

- **人类结果：** 5–10 名目标创作者在 2 周内各完成至少 1 条端到端流水线，且 **≥50%** 表示「比原来少切换工具」或愿意 **每周至少打开 2 次**。
- **商业信号（验证期）：** 至少 **2 人** 明确表示愿为「流水线 + 品牌记忆」付订阅价（具体金额可在验证后定），或愿参与付费内测。
- **下游交付：** 规划阶段无需再发明「第一版做几条流水线、是否自动发帖、是否 MCN 版」等产品行为；未决项仅保留在 Outstanding Questions 的技术/研究类条目。

## Scope Boundaries

### Deferred for later

- 多平台 **自动发布** 与官方 API 深度集成（排期、定时、素材上传）。
- **全平台数据分析** 大屏与自动拉取阅读/粉丝/收益。
- **评论/私信统一收件箱**。
- **表现反馈闭环**（根据历史数据推荐下周选题）——验证留存后再做。
- 用户 **自定义无限流水线**、可视化流程编辑器。
- MCN 多成员权限、结算、达人经纪。
- 内置 **视频剪辑/图像生成引擎**（应外接或调用现有模型服务，非自研编辑器）。

### Outside this product's identity

- 通用 C 端「什么都能做」的超级应用或大盘点式创意清单产品。
- 延续 fastapi-kit **运维向 admin** 作为 C 端主界面（技术栈可复用，产品叙事独立）。
- 替代 ChatGPT / 剪映 / 即梦等 **主力创作工具**（本产品定位为编排与记忆层，第一版在流水线内嵌 AI 步骤，但不承诺全能生成）。
- 面向企业的 **营销自动化/广告投放** 平台。

## Key Decisions

- **产品形态：** 固定流水线 AI 工作台（方案 A），而非纯周计划教练或纯外置记忆层；第一版范围按「2 条流水线 + 品牌记忆 + 发布核对」裁剪。
- **首发流水线：** 短视频（7 步，含分镜/素材/剪辑备注）+ 长图文（7 步，含 SEO/多平台标题变体）；不做「一稿多发自动生成」为第一版核心。
- **商业化边界（验证期主指标）：** 混合计费感知——免费 **2 完整项目/月** + **AI 次数上限**；Pro 提高双上限。
- **证据策略：** 当前依据为行业/竞品调研；在扩功能前必须用 **5–10 人深度试用** 验证，成功标准已写入上文。
- **多平台策略：** 第一版仅 **平台槽位 + checklist**，不做 API 发帖，降低合规与集成成本。
- **技术关系：** 可与 fastapi-kit 同仓作为 C 端后端底座（账号、订阅、异步任务），但不把 B 端运维后台扩展为创作者主产品。

---

## Dependencies / Assumptions

- 目标用户为 **个人/小团队创作者**，非 MCN 企业版优先。
- 主要市场为 **中文内容生态**（平台名称与 checklist 文案可本地化预设）。
- 商业模式假设为 **订阅制**（免费额度 + Pro），具体定价验证期后定。
- AI 能力依赖 **外部模型服务**（自带或用户配置密钥的策略留 planning 决定）。
- 用户已习惯使用至少一种外部 AI 工具；本产品价值是 **减少切换与丢上下文**，而非教育用户第一次用 AI。
